# System consistency

- Follow existing naming conventions for components, variants, and tokens — extend, don't fork.
- Define values as **tokens/variables**; bind components to them rather than hard-coding.
- When you add or change anything, update the components catalog, tokens, and any Code Connect mapping in the same pass.
